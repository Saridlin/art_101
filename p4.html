Exercise 6

In Exercise 6, I created an initial prototype of my final project to test out the libraries that I used: SceneManager, p5.sound, and p5.play. I built the basics of each scene and its buttons and sprites to work on the functionality of the sounds and scene changes first. To navigate through each scenes, the user can click on buttons that take them to a different page or use an associated key. On the Main page, the user can also interact with the isopod sprites and get taken to even more pages where I would put information on each bug in the final version. Through out the program, a soundtrack also plays in the background on loop giving a pleasant atmosphere.
This exercise helped me establish a solid base from which to work on my Project 4 from and made implementing all the stylistic embellishments.

Sound used from XtremeFreddy on Pixabay.

Project 4

For Project 4, I mashed up the ideas of a pet collection and the classic "I-Spy" game with an informative encyclopedia as a theatrification of simulated environment. The user plays by interacting with wiggling sprites of isopods strewn across a forest floor and collects the informative entries about each bug into a journal.
At the opening of the program, the user is greeted with a splash page showing a button to the main screen and a help screen, as well as the title of the game: "I-Spy-Pods!", a play on words of "I-Spy" and "Isopods". A musical track plays on loop in the background until the player clicks into the next screen. At the help screen, a simple summary of the gameplay prompts the player about how to interact with the gaem and lists some key bindings to allow for more movement between screens besides just using the buttons. At the entrance into the main screen, and any screen besides the title, a new music backing track plays. A "click" sound clip also plays when interacting with any of the buttons through out the program as well. At the main screen, the player can then click on any of the wriggling isopod sprites and enter an information page listing some facts about the bug. This interaction is recorded in the journal screen, accessible from the main screen or by pressing 'j'.
The functionality of the screen changes was enabled by SceneManager.js. The p5.sound library allowed for the ambient music and "click" sound clips. And the sprite animations were created using the p5.play library. The button functionality came from a custom checkButtonPress() function from the example code for this project.

I enjoyed creating a fun, interactive program that explores the use of further libraries of p5. I was able to practice my management of pages and scenes within a browser program with each scene having unique events and interactions that occur. The examples given for how each library worked contributed greatly to my understanding of how SceneManager.js worked because it used new function notation I had not worked with before. I believe I created a pleasant program that provides an enjoyable and informative experience to any user. I think I could improve on the organization of my code to make customization and my workflow easier to manage through more custom functions and data structuring. I had implemented some array structures to manage the large amounts of strings and image sprites, but I could have furthered that in the individual bug information page's graphical layouts as well.

Background music from XtremeFreddy on Pixabay.
And the Click Sound Effects are from ZapSplat.
